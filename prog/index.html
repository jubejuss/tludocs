# Loops

  • while

    //while 2**10
let result = 1;
let counter = 0;
while (counter < 10) {
  result = result * 2;
  counter += 1;
}
console.log(result);
//1024

  • do A do loop is a control structure similar to a while loop. It differs only on one point: a do loop always executes its body at least once, and it starts testing whether it should stop only after that first execution. To reflect this, the test appears after the body of the loop.

    // do while 2**10
let result = 1;
let counter = 0;

do {
  counter += 1;
  result += result;
} while (counter < 10);

console.log(result);
//1024

  • for

    // for 2**10
let result = 1;
for (let counter = 0; counter < 10; counter += 1) {
  result = result * 2;
}
console.log(result);
//1024

  • break loops break;

  • length

    console.log(abc.length);
// → 3

  • The last logical operator I will discuss is not unary, not binary, but ternary, operating on three values. It is written with a question mark and a colon, like this:

    console.log(true ? 1 : 2);

    // → 1

    console.log(false ? 1 : 2);

    // → 2

  • raamatust: Automatic type conversion miks ikkagi on sellised konvertimised ja lisaks kohe järgnev materjal ka, ehk see:

    console.log(null == undefined);
// → true
console.log(null == 0);
// → false

  • sama küsimus järgneva kohta - Short-circuiting of logical operators, lihtsalt näidetega selgitust

  • miks sulgudes olevaid muutujaid ei pea deklareerima läbi let'i või var'i või const'i?

    const power = function (base, exponent) {
  let result = 1;
  for (let count = 0; count < exponent; count++) {
    result *= base;
  }
  return result;
};
